## 为什么选择 Next.js

在决定技术栈时，我对比了 Vue/Nuxt、Astro 和 Next.js。最终选择 Next.js 15 App Router 的原因很简单：React 生态最成熟，TypeScript 支持最完善，静态导出模式完美适配我的部署需求。

### 技术栈一览

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 | App Router、SSG、ISR |
| 样式 | Tailwind CSS | 原子化、暗色模式 |
| 组件 | shadcn/ui | 可定制、无依赖锁定 |
| 动画 | Framer Motion | 声明式、性能优 |
| 3D | Three.js (R3F) | React 生态集成 |
| 部署 | Netlify | 自动构建、表单支持 |

## AI 工程化的真实体验

这个项目最大的不同在于开发方式——**全程使用 Claude Code 辅助开发**。

传统的开发流程是：思考 → 编码 → 调试 → 提交，循环往复。而 AI 辅助开发将流程变为：**需求描述 → AI 生成 → 人工审查 → 迭代优化**。

### 几个关键心得

1. **Prompt 是新的代码**：精准的需求描述比技术能力更重要。描述得越清楚，AI 的输出越接近预期。

2. **审查比生成更重要**：AI 生成的代码有 80% 可以直接使用，但剩下 20% 需要人工判断——是否符合项目规范、是否有性能隐患、是否与现有代码一致。

3. **模块化是前提**：AI 擅长在明确的边界内工作。项目架构越清晰、模块越独立，AI 的效率越高。

4. **迭代优于重来**：接受 AI 首次输出不完美，通过多轮对话逐步逼近目标。每一轮迭代都积累上下文，让 AI 越来越懂你的意图。

## 设计系统

平台采用**深空宇宙主题**作为核心视觉语言，搭配玻璃拟态设计风格：

- **色彩**：深黑背景 + 冷白文字 + 淡蓝品牌高亮
- **动效**：星空粒子背景、鼠标光晕追踪、打字机动画
- **交互**：3D 星球模型、卡片悬浮微光、滚动视差
